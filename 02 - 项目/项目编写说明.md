# 准则项目笔记编写说明

本说明定义在本 Vault 中撰写 **IFRS / US GAAP** 会计问题项目笔记的统一格式与流程。  
Cursor Agent 编写时应同时加载 Skill：`.cursor/skills/writing-accounting-standards-notes/SKILL.md`。

**项目总索引**：[[项目索引]]

---

## 一、适用场景

| 场景 | 是否适用 |
|------|---------|
| IFRS / US GAAP 会计处理、披露、分类判断 | ✅ |
| 双准则对比分析 | ✅（C 型） |
| 基于知识库准则原文的分析与决策 | ✅ |
| 需要留档、可复用的实务结论 | ✅ |
| 纯代码开发、Obsidian 配置修改 | ❌ |
| 知识库外准则原文引用 | ❌ |

## 二、文件存放规则

### 目录结构

```
02 - 项目/
├── 项目编写说明.md          ← 本文件
├── 项目索引.md              ← 全部项目笔记 MOC
├── IFRS项目/
│   └── {主题文件夹}/
│       └── {主题}.md
├── USGAAP项目/
│   └── {主题文件夹}/
│       └── {主题}.md
└── 双准则对比/
    └── {主题文件夹}/
        └── {主题}.md
```

### 命名约定

| 项目 | 规则 | 示例 |
|------|------|------|
| 主题文件夹 | 中文，简洁 | `合营联营会计处理/` |
| 文件名 | 与主题一致，中文 | `合营联营定义与会计处理.md` |
| 一篇问题 | 一个文件夹 + 一个主文件 | 不拆成多个碎片文件 |

### 知识库路径（仅允许引用以下目录）

| 准则体系 | 知识库路径 |
|---------|-----------|
| IFRS | `03 - 知识库/IFRS/IFRS准则/`、`03 - 知识库/IFRS/IAS准则/` |
| US GAAP | `03 - 知识库/US GAAP/ASC准则/` |
| 索引 | `03 - 知识库/IFRS/IFRS 知识库总览.md`、`03 - 知识库/US GAAP/US GAAP 知识库总览.md` |

> **红线**：不得引用知识库以外的准则原文；若知识库缺失，在笔记中标注「知识库暂无该准则」，并建议补充下载任务。

## 三、Frontmatter 模板（必选字段）

```yaml
---
tags: [ifrs, ias12, 递延所得税]       # 准则编号 + 主题关键词
date: YYYY-MM-DD
status: active                         # active | archived | needs-update
type: B                                # A | B | C
standards: [IAS 12]                    # 涉及的准则编号列表
related:                               # 关联项目笔记（Obsidian 链接）
  - "[[适用税率选择及计算]]"
---
```

| 字段 | 说明 |
|------|------|
| `tags` | 含准则编号（`ias12`、`asc740`）和问题类型（`披露`、`计量`、`双准则对比`） |
| `status` | `active` 现行；`archived` 已归档；`needs-update` 准则修订后待更新 |
| `type` | A 概念梳理 / B 实务决策 / C 双准则对比 |
| `standards` | 便于 Dataview 筛选与索引维护 |
| `related` | 双向链接相关项目，形成知识网络 |

## 四、正文结构（标准模板）

### 文首 TL;DR（必选）

标题下方、`## 问题` 之前，用 blockquote 写三行摘要：

```markdown
> **TL;DR**：[一句话结论]
> **关键依据**：[核心准则段落]
> **适用场景**：[何时参考本文]
```

### 问题节（必选，结构化）

```markdown
## 问题

### 背景
[事实：架构、交易、科目、管辖区等]

### 决策问题
1. ...
2. ...

### 边界（可选）
[本文不讨论什么]
```

### 通用章节顺序

```markdown
# {标题}

> **TL;DR**：...

## 问题
### 背景 / 决策问题 / 边界

---

## 准则原文（知识库）    # 或「总体框架」+ 分节分析
[引用块 + 提炼表 + Wiki 链接]

---

## 决策分析 / 分析
[表格、对比、逻辑推导]

---

## 结论

### 准则结论
1. ...

### 操作结论          # B 型必选；A/C 型可选
| 情形 | 建议处理 |
|------|---------|

## 实务提示（可选）     # A/C 型推荐
- ...

## 附录（可选）         # 披露模板、Movement 表、附注文字等

---

## 相关项目
- [[...]]

## 准则索引（知识库）
| 准则 | 文件 | 核心段落 |

## 日志
- YYYY-MM-DD：初稿，基于 [准则段落]；变更原因
```

## 五、笔记类型

### A 型：概念梳理

适用：定义、分类框架、会计处理总览。

| 章节 | 必选 | 说明 |
|------|------|------|
| TL;DR | ✅ | 三行摘要 |
| 问题 | ✅ | 背景 + 1–3 个核心问题 |
| 总体框架 | 推荐 | 层级表 / 对比表 |
| 分主题分析 | ✅ | 定义 → 准则原文 → 提炼表 → 处理 |
| 对比总结 / 判断流程 | 推荐 | 表格 + Mermaid |
| 结论（准则 + 操作） | ✅ | 操作结论可选 |
| 实务提示 | 推荐 | 底稿、审计沟通、常见误区 |
| 相关项目 / 准则索引 / 日志 | ✅ | |

**样例**：[[IFRS项目/合营联营会计处理/合营联营定义与会计处理]]

### B 型：实务决策

适用：披露列示、选项选择、具体操作。

| 章节 | 必选 | 说明 |
|------|------|------|
| TL;DR | ✅ | |
| 问题（含选项 A/B） | ✅ | |
| 准则原文 + 提炼表 | ✅ | 按子问题分节 |
| 决策分析 | ✅ | 选项对比、与本公司对照 |
| 准则结论 + **操作结论** | ✅ | 操作结论用表格 |
| 附录（模板/附注文字） | 推荐 | |
| 相关项目 / 准则索引 / 日志 | ✅ | |

**样例**：
- [[IFRS项目/租赁修改披露/租赁修改披露]]

### C 型：双准则对比

适用：IFRS vs US GAAP 差异分析。

| 章节 | 必选 | 说明 |
|------|------|------|
| TL;DR | ✅ | 突出核心差异 |
| 问题 | ✅ | |
| 总体框架（对比表） | ✅ | 机制差异一览 |
| 术语对照表 | 推荐 | 中英 + 准则出处 |
| 分准则分析（US GAAP / IFRS） | ✅ | 各引各的准则 |
| 机制对比 / 判断流程 | 推荐 | |
| 准则结论 + 实务提示 | ✅ | |
| 相关项目 / 准则索引 / 日志 | ✅ | |

**样例**：[[双准则对比/DTA与Valuation Allowance/DTA确认与Valuation Allowance对比]]

## 六、准则引用格式（强制）

### 1. 引用块（区分来源类型）

**英文原文**：

```markdown
> **IAS 12 Paragraph 24（知识库原文）：**
>
> A deferred tax asset shall be recognised...
```

**中文提炼**（来自知识库 `## 📌 中文提炼`）：

```markdown
> **IAS 12（知识库中文提炼）：**
>
> 可抵扣暂时性差异 → 递延所得税资产，须很可能取得应税利润时确认。
```

> 禁止将中文提炼标注为「知识库原文」。

### 2. 引用后提炼表（默认必做）

| 原则 | 原文依据 | 提炼 |
|------|---------|------|
| 确认门槛 | IAS 12 §24 "probable" | 很可能有 taxable profit 才确认 DTA |

### 3. 双向链接

```markdown
[[03 - 知识库/IFRS/IAS准则/IAS 12 - Income Taxes|IAS 12]]
[[03 - 知识库/IFRS/IAS准则/IAS 12 - Income Taxes#1602|跳转至知识库 §1602]]
```

### 4. 术语对照表（C 型及含英文术语时推荐）

| 中文 | 英文 | 准则 |
|------|------|------|
| 估值备抵 | Valuation Allowance | ASC 740 |

## 七、写作风格

| 要求 | 说明 |
|------|------|
| 语言 | 全文中文（准则原文引用块除外） |
| 语气 | 分析型、结论明确 |
| 结构 | `---` 分隔大节；子节用 `###` |
| 表格 | 对比、选项、Movement、提炼 |
| 流程图 | 分类/判断逻辑用 Mermaid `flowchart TD` |
| 结论 | **准则结论**与**操作结论**分层，编号列表 |
| 禁止 | 大段复制知识库全文；无出处的准则主张 |

## 八、Agent 工作流程

### 复杂问题：先输出写作计划（推荐）

```
1. 问题类型：A / B / C
2. 拟引用准则与段落
3. 拟建路径：02 - 项目/{IFRS项目|USGAAP项目|双准则对比}/...
4. 知识库是否覆盖
5. 关联项目笔记
```

### 标准步骤

```
1. 加载本 Skill + 项目编写说明
2. 检索 03 - 知识库/（先中文提炼，再原文段落）
3. 确认知识库覆盖；缺失则标注，不编造
4. 选定类型与目录
5. 写 frontmatter（含 type、standards、related）
6. 写 TL;DR → 问题 → 准则引用 + 提炼表 → 分析 → 分层结论
7. 更新 [[项目索引]]
8. 自检（见下节）
```

## 九、章节速查清单

- [ ] Frontmatter 含 `tags`、`date`、`status`、`type`、`standards`
- [ ] 文首有 **TL;DR**（三行）
- [ ] 「问题」含背景与决策问题（及边界，如适用）
- [ ] 所有准则依据来自 `03 - 知识库/`
- [ ] 引用块标注 `（知识库原文）` 或 `（知识库中文提炼）`
- [ ] 每个主要引用块后有 **提炼表**
- [ ] 结论分 **准则结论** / **操作结论**（B 型操作结论必选）
- [ ] 有 Wiki 链接；C 型有术语对照表（如适用）
- [ ] 有 **相关项目** 链接
- [ ] **项目索引** 已更新
- [ ] 「日志」含日期与变更原因
