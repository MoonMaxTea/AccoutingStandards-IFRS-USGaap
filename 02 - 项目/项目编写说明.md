# 准则项目笔记编写说明

本说明定义在本 Vault 中撰写 **IFRS / US GAAP** 会计问题项目笔记的统一格式与流程。  
Cursor Agent 编写时应同时加载 Skill：`.cursor/skills/writing-accounting-standards-notes/SKILL.md`。

---

## 一、适用场景

| 场景 | 是否适用 |
|------|---------|
| IFRS / US GAAP 会计处理、披露、分类判断 | ✅ |
| 基于知识库准则原文的分析与决策 | ✅ |
| 需要留档、可复用的实务结论 | ✅ |
| 纯代码开发、Obsidian 配置修改 | ❌ |
| 知识库外准则原文引用 | ❌ |

| 项目 | 规则 | 示例 |

## 四、正文结构（标准模板）

按问题类型选用 **A 型（概念梳理）** 或 **B 型（实务决策）**，章节顺序如下。

### 通用必选章节

```markdown
# {标题}

## 问题
[背景 + 编号决策问题]

---

## 准则原文（知识库）
[引用段落 + 双向链接]

---

## 决策分析 / 分析
[表格、对比、逻辑推导]

---

## 结论
[编号结论 + 可操作要点]

---

## 准则索引（知识库）
[Wiki 链接表]

---

## 日志
- YYYY-MM-DD：创建/更新说明
```

### A 型：概念梳理型

适用：定义、分类框架、会计处理总览（如合营 vs 联营）。

| 章节 | 是否必选 | 说明 |
|------|---------|------|
| 问题 | ✅ | 列出 1–3 个核心问题 |
| 总体框架 | 推荐 | 层级表 / 对比表，先给全景 |
| 分主题分析（一、二、三…） | ✅ | 每个概念：定义 → 准则原文 → 处理 |
| 对比总结 | 推荐 | 合营 vs 联营、IFRS vs US GAAP |
| 判断流程 | 推荐 | Mermaid flowchart |
| 结论 | ✅ | 3–5 条编号结论 |
| 准则索引 | ✅ | |
| 日志 | ✅ | |

**参考样例**：[[合营联营会计处理/合营联营定义与会计处理]]

### B 型：实务决策型

适用：具体披露列示、税率选择、修改会计处理等。

| 章节 | 是否必选 | 说明 |
|------|---------|------|
| 问题 | ✅ | 背景事实 + **决策问题**（选项 A/B 或编号列表） |
| 准则原文（知识库） | ✅ | 按子问题分节引用 |
| 决策分析 | ✅ | 准则要求表、选项对比、与本公司对照 |
| 结论 | ✅ | 完整结论陈述 + **本公司操作结论** |
| 模板/附表 | 可选 | Movement 表、披露模板等 |
| 文档版本信息 | 可选 | 版本号、负责人（实务项目用） |
| 准则索引 | 推荐 | |
| 日志 | ✅ | |

**参考样例**：
- [[租赁修改披露/租赁修改披露]]
- [[所得税披露/适用税率选择及计算]]

## 五、准则引用格式（强制）

### 1. 段落引用块

```markdown
> **IFRS 11 Paragraph 7–8（知识库原文）：**
>
> Joint control is the contractually agreed sharing of control...
```

```markdown
> **IAS 12 Paragraphs 1602-1612（知识库原文）：**
>
> In explaining the relationship between tax expense...
```

```markdown
> **ASC 842-20-35-1（知识库原文）：**
>
> ...
```

**格式要点**：
- 标注准则编号 + 段落号 + `（知识库原文）`
- 引用块内保留英文原文（知识库中为英文）
- 引用块后可用「中文提炼」或「核心原则提炼」表格

### 2. 双向链接

```markdown
[[03 - 知识库/IFRS/IFRS准则/IFRS 11 - Joint Arrangements|IFRS 11]]
[[03 - 知识库/IFRS/IAS准则/IAS 28 - Investments In Associates And Joint Ventures|IAS 28]]
[[03 - 知识库/US GAAP/ASC准则/ASC 842 - Leases|ASC 842]]
```

段落锚点（若知识库有行号）：

```markdown
[[03 - 知识库/IFRS/IAS准则/IAS 12 - Income Taxes#1602|跳转至知识库 §1602]]
```

### 3. 提炼表格

引用原文后，用表格归纳：

| 原则 | 原文依据 | 提炼 |
|------|---------|------|
| 选择标准 | §1605 "most meaningful information" | 向使用者提供最有意义的信息 |

## 六、写作风格

| 要求 | 说明 |
|------|------|
| 语言 | 全文中文（准则原文引用块除外） |
| 语气 | 分析型、结论明确，避免模糊表述 |
| 结构 | `---` 分隔大节；子节用 `###` |
| 表格 | 优先用表格呈现对比、选项、Movement |
| 流程图 | 分类/判断逻辑用 Mermaid `flowchart TD` |
| 结论 | 必须编号（1. 2. 3.），可执行 |
| 禁止 | 大段复制知识库全文；无出处的准则主张 |

## 七、Agent 工作流程

收到 IFRS / US GAAP 问题时的标准步骤：

```
1. 明确问题类型（A 概念 / B 决策）
2. 在 03 - 知识库/ 中检索相关准则（Grep / 索引）
3. 确认知识库是否有所需准则；缺失则标注并停止编造
4. 选定目录：02 - 项目/IFRS项目/ 或 USGAAP项目/
5. 按本说明第四节结构起草笔记
6. 准则原文逐段引用，附 Wiki 链接
7. 写出编号结论；B 型补充「本公司操作结论」
8. 填写日志
```
